# Claude Chat Logger

Record and archive Claude Code conversations as readable Markdown files with enhanced formatting.

## What It Does

A pair of slash commands (`/session_record_start` + `/session_record_done`) that capture a specific segment of your Claude Code conversation and archive it as a well-formatted Markdown file.

**Key features:**
- On-demand segment extraction (not the whole transcript, just what you mark)
- Merged Claude blocks (consecutive tool calls + text response in one block)
- Enhanced tool call display (search queries, URLs, file paths, user selections)
- Turn numbering (#1, #2, #3) with TOC and anchor links
- Session stats footer (duration, message counts, tool usage, files touched)
- Works across context compaction (reads from JSONL transcript on disk)

## File Structure

```
commands/
  session_record_start.md   # Slash command: marks recording start
  session_record_done.md    # Slash command: ends recording, triggers archival
scripts/
  session-finalize.mjs      # Node.js archival script
```

## Installation

Copy the files to your Claude Code config directory:

```bash
# Commands (slash commands)
cp commands/session_record_start.md ~/.claude/commands/
cp commands/session_record_done.md ~/.claude/commands/

# Scripts
cp scripts/session-finalize.mjs ~/.claude/scripts/
chmod +x ~/.claude/scripts/session-finalize.mjs
```

**Requirements:** Node.js (for `session-finalize.mjs`). No npm packages needed.

## Usage

1. In any Claude Code conversation, type `/session_record_start` (optionally with a topic)
2. Have your conversation normally
3. When done, type `/session_record_done`
4. Claude generates a summary, and the script archives everything to `docs/sessions/`

## How It Works

1. `/session_record_start` leaves a timestamp marker in the JSONL transcript
2. `/session_record_done` tells Claude to generate a summary, then pipes it to the finalize script
3. `session-finalize.mjs` reads the JSONL transcript, finds the start/done boundary, extracts messages, merges consecutive Claude blocks, renders enhanced Markdown, computes stats, generates TOC, and writes the archive file + updates `INDEX.md`

## Architecture (session-finalize.mjs)

```
parseStdin()        -> { slug, title, summary, body }
validateSlug()      -> blacklist check
findTranscript()    -> most recent .jsonl
findBoundary()      -> { startTs, endTs, sessionType }
extractMessages()   -> [Message] with tool calls + results
buildTurns()        -> merge consecutive assistant msgs into turns
renderTimeline()    -> enhanced Markdown with anchors
computeStats()      -> duration, counts, tool usage
generateTOC()       -> Markdown TOC linking to #turn-N
writeOutput()       -> file + INDEX.md update
```
